# ストーリー生成機能 実装計画

## 概要
家族の日常写真から自動でストーリー（絵本・漫画・新聞）を生成する機能を実装する。

## 目標
- Google Photosとの差別化を明確化
- 家族の「話題作り」を自動化
- 子供の日常をエンタメコンテンツに変換

---

## Phase 1: MVP（最小実装可能製品）

### 1.1 API設計

#### 新規エンドポイント: `/api/create-story`

**リクエスト**:
```typescript
{
  images: File[];           // 1〜5枚の写真
  storyType: "picture-book" | "comic" | "newspaper";
  language: "ja" | "en";    // 出力言語
  tone: "funny" | "cute" | "adventure";  // ストーリーのトーン
}
```

**レスポンス**:
```typescript
{
  story: {
    title: string;
    panels: Array<{
      imageIndex: number;
      caption: string;
      dialogue?: string;
    }>;
    summary: string;
  };
  generatedImage?: string;  // 合成画像（Base64）
}
```

#### Gemini APIプロンプト設計

```typescript
const STORY_PROMPT = `
あなたは家族の写真から面白いストーリーを創作するアシストです。

## 指示
1. アップロードされた写真の人物、表情、状況を分析してください
2. {storyType}形式でストーリーを作成してください
3. トーンは{tone}で、家族が楽しめる内容にしてください
4. セリフやナレーションは{language}で出力してください

## 出力形式
JSON形式で以下の構造で出力してください：
{
  "title": "ストーリーのタイトル",
  "panels": [
    {
      "imageIndex": 0,
      "caption": "このパネルの説明文",
      "dialogue": "キャラクターのセリフ（任意）"
    }
  ],
  "summary": "ストーリーの要約"
}

## 制約
- パネル数は画像の枚数と同じにしてください
- 各パネルのキャプションは50文字以内にしてください
- セリフは自然な会話にしてください
- 家族に不適切な内容は避けてください
`;
```

## Phase 2: 機能拡張

### 2.1 合成画像生成
- パネルを1枚の画像に合成
- 絵本風レイアウト（絵本形式）
- 4コマ漫画レイアウト（漫画形式）
- 新聞風レイアウト（新聞形式）

### 2.2 音声読み上げ
- ストーリーを音声で読み上げ
- 子供向けのゆっくりしたスピード
- キャラクターごとに声色を変更

### 2.3 ストーリー保存機能
- 過去のストーリーをカレンダー形式で閲覧
- 「1年前の今日」のストーリーを自動表示
- 家族内での「お気に入り」投票機能

---

## Phase 3: 家族特化機能

### 3.1 家族メンバープロフィール
- 家族の名前、呼び名、誕生日を登録
- ストーリー内で自動的に名前を使用
- 「〇〇ちゃんの成長記録」を自動整理

### 3.2 対話型思い出アシスタント
- 写真について質問できるチャットUI
- 「このとき何してたっけ？」という質問にAIが回答
- 子供向けの「やさしい日本語」モード

### 3.3 知育コンテンツ生成
- 間違い探し、迷路、英単語カードを自動生成
- お気に入りのおもちゃや絵が教材になる
- 市販の知育プリントより子供の食いつきが良い

### リスク1: Gemini APIのコスト

### リスク3: パフォーマンス問題
- **対策**: 画像の最適化、CDNの活用、非同期処理
